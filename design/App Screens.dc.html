Claude Code Workflows
Hooks & lifecycle events
~6 min read
Hooks are user-defined shell commands that run at fixed points in Claude Code's lifecycle. Unlike instructions in CLAUDE.md — which the model may or may not follow — hooks are deterministic: they always execute.
The two most-tested events are PreToolUse, which fires before a tool call and can block it, and PostToolUse, which fires after the tool completes.
// .claude/settings.json
"hooks": { "PreToolUse": [{
  "matcher": "Bash",
  "hooks": [{ "type": "command",
    "command": "./check.sh" }] }] }
Exam key point
A PreToolUse hook exiting with code 2 blocks the tool call and feeds stderr back to Claude. Any other non-zero exit shows the error to the user only.

Question 6 of 15 · D4
⚑
A team wants Claude to query their internal ticketing system from multiple surfaces — Claude Code, the desktop app, and a custom agent. What is the recommended integration approach?
A · Paste API docs into each conversation
B · Build one MCP server exposing ticket tools✓
C · Write a custom tool in each client✕ yours
D · Fine-tune a model on ticket data
Explanation
MCP exists precisely for this: one server, written once, gives every MCP-capable client the same tools. Per-client custom tools (C) duplicate work and drift apart; docs-pasting (A) doesn't execute anything; fine-tuning (D) doesn't grant live data access.

D5 · Context Management
What happens during context compaction in Claude Code?
The conversation is summarized and the summary replaces the full history, freeing context window space while preserving key facts, decisions and open tasks. Triggered automatically near the limit, or manually with /compact.
